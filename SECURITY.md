# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Rocket, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Private Vulnerability Reporting** — Use the "Report a vulnerability" button on the [Security tab](https://github.com/mboudriga/rocket/security) of this repository.
2. **GitHub Discussions** — If you are unsure whether something is a vulnerability, start a private thread in [GitHub Discussions](https://github.com/mboudriga/rocket/discussions).

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment** — Within 48 hours of your report.
- **Assessment** — An estimated timeline for a fix within 7 days.
- **Resolution** — You will be notified when the vulnerability is fixed.
- **Credit** — You will be credited in the release notes unless you prefer to remain anonymous.

## Scope

This policy applies to the Rocket monorepo and all packages under the `@rocket/*` scope.
