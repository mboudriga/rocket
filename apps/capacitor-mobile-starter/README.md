# Capacitor Mobile Starter

Cross-platform mobile app template using **Capacitor 8** for iOS and Android. Built for business/enterprise apps with native device features.

## Stack

- **React 19** + **TypeScript** + **Vite**
- **Capacitor 8** ŌĆö iOS + Android native runtime
- **react-router-dom v7** ŌĆö BrowserRouter (History API works with Capacitor)
- **@rocket/ui** ŌĆö UI components
- **Zustand** ŌĆö Client state

## Prerequisites

- Node.js 24+, pnpm 10+
- **iOS**: macOS + Xcode 16+
- **Android**: Android Studio + JDK 17+

## Getting Started

```bash
# Install dependencies (from monorepo root)
pnpm install

# Start dev server (web preview)
pnpm --filter @app/capacitor-mobile-starter dev

# First-time native setup
cd apps/capacitor-mobile-starter
npx cap add ios
npx cap add android
```

## Scripts

| Command | Action |
|---------|--------|
| `pnpm dev` | Start Vite dev server on :3008 |
| `pnpm build` | Production build to `dist/` |
| `pnpm cap:sync` | Sync web build to native projects |
| `pnpm cap:ios` | Open iOS project in Xcode |
| `pnpm cap:android` | Open Android project in Android Studio |
| `pnpm cap:run:ios` | Build & run on iOS simulator |
| `pnpm cap:run:android` | Build & run on Android emulator |
| `pnpm typecheck` | TypeScript type checking |
| `pnpm lint` | Biome lint |
| `pnpm clean` | Remove dist/, ios/, android/ |

## Dev Workflow

1. `pnpm dev` ŌĆö develop in browser at http://localhost:3008
2. `pnpm build && pnpm cap:sync` ŌĆö sync to native
3. `pnpm cap:run:ios` or `pnpm cap:run:android` ŌĆö test on device/simulator

For live reload on device, Capacitor uses the Vite dev server URL via `--host`.

## Native Platform Setup

### iOS ŌĆö Required Info.plist Entries

After `npx cap add ios`, add these to `ios/App/App/Info.plist` (Capacitor does NOT add them automatically ŌĆö missing entries crash the app):

```xml
<key>NSCameraUsageDescription</key>
<string>This app uses the camera to take photos</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>This app accesses your photos</string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string>This app saves photos to your library</string>
<key>NSLocationWhenInUseUsageDescription</key>
<string>This app uses your location</string>
```

### Android ŌĆö Required Manifest Permissions

After `npx cap add android`, verify these exist in `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
```

Camera permissions are handled automatically by the Capacitor Camera plugin.

### Capacitor 8 SystemBars

Capacitor 8 introduces a **SystemBars** core plugin (bundled in `@capacitor/core`) that replaces the older `@capacitor/status-bar` for edge-to-edge apps. Configure in `capacitor.config.ts`:

```typescript
plugins: {
  SystemBars: {
    style: 'dark',        // 'dark' | 'light'
    backgroundColor: '#00000000',  // transparent
  },
}
```

### Live Reload on Device

To use live reload, temporarily add `server.url` to `capacitor.config.ts`:

```typescript
server: {
  androidScheme: 'https',
  url: 'http://YOUR_LAN_IP:3008',  // e.g., http://192.168.1.68:3008
  cleartext: true,
},
```

Then run `npx cap copy && npx cap run ios` (or `android`). Remove `url` and `cleartext` before production builds.

## Notes

- `ios/` and `android/` are generated by `npx cap add` and gitignored
- Capacitor 8 defaults `androidScheme` to `https` ŌĆö History API routing works on both platforms
- The `--host` flag in dev exposes the server on the network for device testing
- Capacitor 8's `npx cap add ios` defaults to **Swift Package Manager** (not CocoaPods)
